# Contributing to Projects on GitHub

Thank you for your interest in contributing! Follow the steps below to contribute to any project on GitHub.

---

## 🛠 Prerequisites

- A [GitHub account](https://github.com/join)
- [Git](https://git-scm.com/downloads) installed on your local machine
- Basic familiarity with the command line

---

## 🚀 Step-by-Step Contribution Guide

### 1. Fork the Repository

Click the **Fork** button at the top-right of the repository page to create your own copy of the project.

### 2. Clone Your Fork

```bash
git clone https://github.com/<your-username>/<repository-name>.git
cd <repository-name>
```

### 3. Create a New Branch

Always create a new branch for your changes instead of working on `main`:

```bash
git checkout -b feature/your-feature-name
```

### 4. Make Your Changes

- Follow the existing code style and conventions of the project.
- Keep changes focused and minimal — one feature or bug fix per pull request.
- Add or update tests if applicable.

### 5. Commit Your Changes

Write clear, descriptive commit messages:

```bash
git add .
git commit -m "feat: add short description of your change"
```

### 6. Push Your Branch

```bash
git push origin feature/your-feature-name
```

### 7. Open a Pull Request (PR)

1. Go to the original repository on GitHub.
2. Click **Compare & pull request**.
3. Fill in the PR title and description explaining **what** you changed and **why**.
4. Submit the pull request and wait for a review.

---

## ✅ Contribution Checklist

Before submitting a PR, make sure you have:

- [ ] Forked the repository and created a feature branch
- [ ] Made focused, minimal changes
- [ ] Written a clear commit message
- [ ] Pushed your branch and opened a pull request
- [ ] Described your changes clearly in the PR description

---

## 💬 Need Help?

Feel free to open an [issue](../../issues) if you have questions or need guidance before starting your contribution.
